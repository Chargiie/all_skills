# 旅游攻略手机 PDF Skill（屏卡模型）

把旅游攻略渲染成**手机竖屏、可分享**的 PDF。本文件管**流程、工具链、机械契约、验证门**；**视觉样式不在这里约束，留给模型自由发挥**——调性、配色、版式按场景自己定，**不要照搬模板**。（`docs/style-reference.md` 是配色 / 密度数值 / 组件的**菜单**，没思路或要密度锚点时翻一下，**别照抄**——照规定式参考做容易出模板化死板的产物。）

## 核心单位：屏卡（screen-card）
**每一页 = 一张屏卡 = 一屏（400×890 CSS px，9:20）的、自洽完整、可独立阅读 / 截屏分享的设计单元。** 整份攻略 = 一叠屏卡按序成册；**单张卡也能独立作为产物**。

这是本 skill 的根本范式，区别于「写一坨流动 HTML 让浏览器自动分页」：
- **不靠自动流动分页**——那会让标题成孤儿、内容贴边、模块被切。**靠按卡编排内容**：你决定每张卡放什么。
- **固定一屏高是对的**（`.screen{height:890px}`，1 卡 = 1 页）。固定高度的老坑（流动内容撞固定高被 `overflow:hidden` 静默裁掉、多页塌成一页）这里靠**渲染期溢出检测 + 逐卡看图**根治，不是靠回避固定高度。
- **每张卡必须完整自洽**：单拎出来看不缺不残、信息够读。

## When to use
- 旅游攻略 / citywalk / 周末出游 / 多日行程，出口是**手机**、要 **PDF 或可分享卡片**
- **不适用**：纯文本回答、桌面 A4 报告、数据驱动可复现文档（那走 reportlab）

## 技术栈
HTML → Chromium(Playwright) → PDF。强视觉攻略用 HTML+Chrome 最顺手；**不要**走 LaTeX/reportlab。

## 工作流程
1. **需求拆解**：人数 / 天数 / 出发地 / 预算 / 节奏 / 必去点 / 硬性要求（如「两套路线」「机酒价」）。
2. **信息核查**：`web_search` 查真实店名 / 民宿 / 景点 / 价格区间，**禁编造**；无 live 价时标「估算 + 时间」。配图同理——只用**验证过能打开**的 URL（见机械契约「图片真实可达」）。
3. **内容分卡**：把内容编排进若干张屏卡，**每卡一个明确角色**（封面卡 / 路线卡 / 某点位卡 / 贴士卡 / 预算卡…）。每卡目标**自洽完整**：自带标题或定位标识，单拎出来也读得懂。先想清楚「这份攻略是几张卡、各装什么」，再写代码。
4. **每卡定密度档 + 适配排版**：估这张卡的内容量 → 挑密度档，按档调字号 / 行距 / 间距 / 布局（**数值锚点查 `style-reference.md` 密度菜单**）：
   - **稀疏**（封面 / 章节卡 / 单点 hero）：少元素 → 大字、大留白、居中焦点、一个视觉主角。
   - **均衡**（标准点位卡 / 2–3 块）：中字号、舒适间距、卡片化。
   - **密集**（路线时间轴 / 长清单 / 对照表）：紧凑字号行距（但守可读下限）、小间距、必要时分栏。
   - **同一张卡内容多就上密集档、少就上稀疏档——参数跟着内容量走，别一套参数套全程。**
5. **写 HTML**：每卡一个 `<section class="screen">`（`height:890px`，满版）。渲染器靠 `.screen` 类名做护栏，**务必用这个类名**。
6. **渲染 PDF**：`node scripts/render_mobile.cjs source.html out.pdf`（需 `NODE_PATH` 指向装了 playwright 的 node_modules）。读它输出的**逐卡溢出 / 填充报告**。
7. **逐卡自检**：过下方验证门（渲染报告 + 看图）。
8. **交付**：整份 PDF / 单卡 PDF 均可（少放几张 `.screen` 就是单卡或精简册）。

## 机械契约（渲染正确性，非样式偏好）
不照做会出坏产物，和审美无关，必须遵守：

**屏卡结构**
- **一卡一屏一页**：每张卡 `<section class="screen">`，`height:890px`（= 300pt×667.5pt；**Playwright `page.pdf` 不收 `pt`**，渲染器用 `width:'400px' height:'890px'`）。`.screen` 内用 `display:flex; flex-direction:column` 排，留白靠 CSS 内边距。
- **渲染页数必须 == 你编排的卡数**。不等 → 某张卡内容超 890px 被迫拆页（密度过密）→ **降档或拆卡**，别放任。
- **每卡完整性（硬要求）**：① 无文字 / 元素在卡边界被裁切或出血截断；② 填充度约 **60–100%**，不留大片空白凑页（有意的 hero 留白除外，但要看图确认是「设计」不是「没做完」）；③ **单卡自洽**——带标题 / 定位标识，抽任一页单独看信息完整。
- **满版无白边**：渲染 `margin:0`、`displayHeaderFooter:false`；页边距 / 留白全靠 **CSS 内边距**，别靠渲染器 margin。背景色铺到 `html, body` 和 `.screen`，**翻页处不能出现白条 / 断裂**。

**内容真实**
- **链接必须真可点**：地图 / 订房 / 参考用真实 `<a href>`（渲染成 PDF `/Link`），**禁假链接 div**。
- **图片真实可达**：配图只用**验证过能打开**的 URL，别凭记忆编文件名（实测：编造的 Wikimedia 文件名全 404 留白）。Wikimedia 走 Commons API 搜真实文件名 → `imageinfo` 取 `thumburl`；任意图床嵌入前 `curl -sI <url>` 确认 200。**宁可少图 / 用内联 SVG，不留 404 空块**。

**渲染细节**
- **不横向滚动**：内容宽 ≤ 卡内容区宽；长 URL / 表格换行或缩短。
- **CJK 字体栈 / 回退**：`-apple-system, "PingFang SC", "STSong", "Helvetica Neue", Arial, sans-serif`，回退顺序 `SC→JP→KR→Latin`（日文在前会让简体用日文字形），字型最多两种；webfont 截图前 `await document.fonts.ready`。
- **颜色保真**：`print-color-adjust: exact` 挂 `*`（挂 body 无效）。
- **图表少用**：攻略一般不用图表；万一用 Chart.js，必须 `animation:false` 且等渲染完再截。

## 渲染器（scripts/render_mobile.cjs）
满版无白边（margin 0，留白交给 CSS），并**对每张 `.screen` 做溢出 / 填充检测**：任一卡内容超一屏（会被裁切 / 出血）→ 退出码 1 + 指出是哪几张卡；偏空的卡给 warn。无 `.screen` 时回退普通流动渲染（向后兼容，但护栏不生效）。用法：
```bash
NODE_PATH=<装了 playwright 的 node_modules> node scripts/render_mobile.cjs source.html out.pdf
```
输出 JSON 含 `screens`（卡数）、逐卡 `contentH/fill/overflow`、`overflow_cards`、`underfill_cards`。**`overflow_cards` 非空必须改到空再交付。**

## 验证门（流程纪律，缺一不可）
| 检查 | 命令 / 方法 | 通过标准 |
|---|---|---|
| 页数 == 卡数 | `pdfinfo out.pdf \| grep Pages` 对比编排卡数 | 相等；多出来 = 有卡溢出被拆页 |
| 无卡溢出 / 裁切 | 看 `render_mobile.cjs` 报告 | `overflow_cards` 为空、退出码 0 |
| 填充合理 | 渲染报告 `fill` + 看图 | 各卡约 60–100%；偏空的卡确认是有意 hero 否则补 |
| 单卡自洽 | 看图：抽任一页单独看 | 不缺不残、有标题 / 定位、信息可读 |
| 图片未丢 | `pdfimages -list out.pdf \| tail -n +3 \| wc -l` | ≥ 预期数（404 会使计数 < 预期 → 漏图） |
| 链接可点 | `pypdf` 数 `/Link` + spot-click ≥3 | 命中、能跳转 |
| 文字可读 | `pdftotext -layout` | 无乱码、无残留占位符 |
| 翻页连续 | 看图 | 背景跨页连续、无白条断裂 |

> **看图是主验证手段**：渲染后必跑 `pdftoppm -r 96 -png out.pdf /tmp/pg`，用 **Read 逐页看图**——每页都该是一张完整屏卡：无裁切、无大白凑页、单看自洽。渲染报告能自动报溢出，但贴边 / 留白美感 / 单卡是否成立**只有看图能终判**。
>
> `pdftotext` 看不到图像，单独用会**静默放行无图 PDF**——必须配 `pdfimages`。任何步骤 **MUST NOT** `2>/dev/null`（丢掉唯一的失败信号）。

调性、配色、卡片 / 标签 / callout / emoji、字号行距——**模型按场景与每卡密度档自由发挥，不要照搬任何模板**。只要过上面的机械契约和验证门即可。
